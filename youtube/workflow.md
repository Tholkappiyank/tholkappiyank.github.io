# Workflow — TubeVault / TholsStudio

How the static site is built, regenerated, and maintained. Everything is
plain Node / browser — **no build dependencies**.

---

## File map

| File | Role |
|------|------|
| `page-template.html` | Single source of truth for page markup. Uses `{{APP_NAME}}`, `{{APP_LABEL}}`, `{{APP_COLOR}}`, `{{JS_DIR}}`, `{{SCRIPT_TAGS}}` placeholders. |
| `categories.json` | Category manifest — the list of categories (`id`, `name`, `color`). Read by both `build.js` and `maintain.html`. |
| `build.js` | Node script: reads `page-template.html` + `categories.json` and **writes** all pages into the repo. |
| `delete-category.js` | Node script: removes a category's `<id>.html` + `<id>/` folder. |
| `app.js` / `style.css` | Shared engine, loaded by every category page. |
| `maintain.html` | Optional browser-only generator (reads `categories.json` over HTTP). |
| `*-collections.js` / `*-playlist.js` / `*-watched.js` | Per-category seed data, created in the app, committed inside `<id>/`. |
| `tests.html` / `tests.js` | Feature-test harness (regenerated by `build.js`). |

---

## Canonical workflow (terminal)

```bash
cd youtube

# 1. Edit the template (layout / styles / markup) in page-template.html
# 2. Add / remove a category in categories.json
# 3. Regenerate everything:
node build.js

# 4. (optional) Remove a category entirely:
node delete-category.js <id> --force
```

`build.js` writes:

```
music.html            ← root
electronics.html
astrology.html
tests.html
music/page-template.js
electronics/page-template.js
astrology/page-template.js
test/page-template.js
```

The per-category seed files (`<id>-collections.js`, `-playlist.js`,
`-watched.js`) are **not** generated by `build.js` — they are produced in the
app (see "Seed data" below).

Commit & push. GitHub Pages serves the `youtube/` folder.

`build.js` derives its page list from `categories.json` (via
`loadCategories()`); only the `test` page is hardcoded, because it is the
feature-test harness, not a user category.

## Add a new category

1. Append an entry to `categories.json`:
   ```json
   { "id": "cooking", "name": "Cooking", "color": "#FFA726" }
   ```
2. `node build.js` → creates `cooking.html` + `cooking/page-template.js`.
3. Open `cooking.html` in the app → use **Create category…** (or copy another
   category's seeds) to produce `cooking/cooking-collections.js` etc. Commit them.
4. To keep the in-app switcher in sync, also add the same entry to
   `BUILTIN_APPS` in `app.js` (see "Caveats").

## Remove a category

```bash
node delete-category.js cooking --force   # deletes cooking.html + cooking/
```

`delete-category.js` refuses the built-ins (`music` / `electronics` /
`astrology`) and `test`; to remove a built-in, edit `categories.json` first.
If the category was created in the app, also clear it from the browser
`localStorage` registry (`tholsstudio_categories`) via the in-app category
switcher.

## Browser-only workflow — `maintain.html`

Serve the folder over HTTP, then open `maintain.html`:

```bash
python3 -m http.server        # then visit http://localhost:8000/maintain.html
```

- Loads `page-template.html` and `categories.json` via `fetch` (HTTP only;
  on `file://` it falls back to a built-in category list and logs a warning).
- Add / Remove rows in the table, then **Generate** downloads `<id>.html` +
  `page-template.js` to your Downloads folder — copy them into the repo manually.
- It can only **download**, never delete. Use `delete-category.js` for removal.

## Seed data

The actual videos/playlists/watched history live in the per-category seed
files, not in the template or `categories.json`:

```
<id>/
  <id>-collections.js   → const DEFAULT_COLLECTIONS = [ { id, name, color, groups, ungrouped } ]
  <id>-playlist.js      → const DEFAULT_PLAYLISTS = [ ... ]
  <id>-watched.js       → const WATCHED_VIDEO_IDS = [ ... ]
  page-template.js      → window.PAGE_TEMPLATE = "<!DOCTYPE …>"
```

These are created inside the app (Add Video / Import / Create category…) and
committed. `build.js` only writes `page-template.js` into each `<id>/` folder;
the other three are source data you supply.

`categories.json` is intentionally just a **category manifest** (`id` / `name`
/ `color`) — it does not and should not carry group/video data, since
`build.js` and `maintain.html` ignore those fields and they would be misleading.

## Fresh deploy (no categories shipped)

```bash
node delete-category.js music --force
node delete-category.js electronics --force
node delete-category.js astrology --force
```

Keep `app.js`, `style.css`, `page-template.html`, `categories.json`,
`tests.html`. Visitors create their own categories in-app via
"Create category…".

---

## Caveats

- **`app.js` has a separate hardcoded `BUILTIN_APPS`.** The browser engine must
  work over `file://`, where it cannot `fetch` `categories.json`. When you
  change `categories.json`, also update `BUILTIN_APPS` in `app.js` if you want
  the in-app category switcher to match the built pages.
- **`build.js` and `maintain.html` both read `categories.json`**, so they stay
  in sync with each other.
- **Open `tests.html` after a build** to confirm the feature suite still passes.
